4.  Description of the module of SCT rpm

4.1. Short

SCT rpm is a manager of packages. Intended for work with rpm packages.

4.2. Description of SCT rpm.

Package manager of SCT rpm The groups of bales are placed in the left panel of manager, at the choice of concrete group of bales bales which belong to this group will be represented in right overhead part. At the choice of concrete bale down in pracilia of panel information will be represented about this bale. Accessible operations which can be conducted above bales accessible in the menu of rpm :

  • install is installation of the chosen packages.

  • remove is a delete of the chosen packages.

  • find is a search of the chosen packages.

  • edit source – the manager of sources loads. (details see a section Manager of sources)

There are commands in the menu of view:

  • installed packages – to show all the packages are set.

  • not installed packages – to show all the packages are not set.

  • update – update information about packages, to take off a selection from all packages.

4.3. Set and delete of packages.

That to set the chosen packages it is necessary to choose a command rpm/install (rpm/remove – if case of delete), or push the button of install (in the case install) or remove (in the case of delete) down on the toolbar. Master of install will be loaded.

The program automatically will check up dependences of the packages chosen by you and will add those which were not chosen by you, however, which are needed for satisfaction of dependences of the chosen packages. Packages are chosen by you will finds' in the section of selected packages added the program in the section of additional packages.

Important
program conducts a search only in the indicated sources (div section of certificate manager of source)

. In case if the program will add packages, a section will appear the Additional packages, two variants will be offered to you:

  • to set(to delete) all – packages will be set(remote) indicated in sections chosen and additional.

  • to set(to delete) only chosen – packages which are indicated in a section Chosen will be set(remote), in this case there can be negative consequences (and can be not): some programs can be not loaded, or work incorrectly, in the case of delete the choice of this option can lead to collapse of the operating system.

Important
that near every bale a mark stood in a report – it is completed, it means that dependences are decided, if not it means that the program was not able to find a necessary package, or file in a source, to satisfy dependences. At that rate you must find a package is needed and to add in a source and repeat installation. you also can ignore it and prolong establishment, but then you will add the system a risk described higher.

4.4. Find packages

To find a package or file it is necessary to choose a command Rpm/Find, to push the button of find on the toolbar. In the field of enter the name of package or file it is then necessary to choose a type: that it is needed to find a package, whether a file is in a package, and to push the button of find. Also accessible options: absolute coinciding, with an account a register; search in: name, description, result.

4.5. Source manages.

You can execute various operations with sources. A source is this place (folder, cd, dvd) where rpm packages are. Accessible next operations of addition, delete, editing, renewal.

  • Addition is addition of new source.
  • Delete is a delete of existent source.
  • Editing is a change of the name of existent source, or to the way where it is.
  • Update – if you to the source (for example, to the catalogue ) added new packages or deleted about it it is needed to report manager of sources, choosing a necessary source and pushing the button to renew.

To save information about added, remote or press the renewed sources button of save, if no - discard.

4.6. Wizard addition and editing of source

Enter name of source (choose on own discretion), option of cd/dvdset on te that this source is Cd/dvd by a disk. In path source specify where packages are. Push button of next, to begin the process of addition. Given about the added sources going in the file of sct/share/heder.xml.
